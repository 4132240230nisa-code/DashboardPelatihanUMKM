Skenario Validasi (6 Teknik)
🔴
Test for Missing Data
V-01

Deteksi field kosong sebelum data dikirim. Semua field wajib (*) divalidasi saat submit dan saat focus hilang (on-blur).

❌ Error Message
"Field NIK wajib diisi."
"Field Nama Lengkap wajib diisi."
"Field Nomor HP wajib diisi."
✅ Mekanisme
Sistem memblokir submit → highlight border merah → scroll ke field pertama yang kosong → tampilkan pesan error spesifik per field.
SkenarioSistem
NIK dikosongkan, klik SubmitREJECT "NIK wajib diisi"
Semua field terisiPASS Lanjut validasi
📏
Test for Correct Field Length
V-02

Validasi panjang karakter untuk setiap field dengan aturan minimal dan maksimal.

⚠ Aturan Panjang
NIK: tepat 16 digit
Nama: min 3, max 80 karakter
No. HP: min 10, max 13 digit
Alamat: min 10 karakter
Kode Peserta: tepat 8 karakter (TNG-XXXX)
❌ Error Message
"NIK harus tepat 16 digit (saat ini: 14 digit)."
"Nama terlalu pendek — minimal 3 karakter."
"Nomor HP harus 10–13 digit."
InputRespons
NIK = "12345678" (8 digit)REJECT panjang salah
NIK = 16 digit tepatPASS
HP = "0812" (4 digit)REJECT terlalu pendek
🔤
Test for Class / Composition
V-03

Validasi tipe data dan komposisi karakter — apakah input sesuai kelas yang diharapkan (angka saja, huruf saja, alfanumerik, dll).

📐 Aturan Tipe
NIK → angka saja (regex: /^\d+$/)
Nama → huruf & spasi saja
No. HP → angka, boleh awali +62
Omzet → angka positif
Kode → TNG-[0-9]{4}
❌ Error Message
"NIK hanya boleh berisi angka. Karakter '#' tidak diizinkan."
"Nama tidak boleh mengandung angka atau simbol."
"Format kode peserta salah. Gunakan TNG-XXXX."
InputRespons
NIK = "36710-12509-001" (ada strip)REJECT bukan angka
Nama = "Budi123"REJECT ada angka
Kode = "TNS-1234"REJECT bukan TNG
📊
Test for Range / Reasonableness
V-04

Periksa apakah nilai berada dalam rentang yang masuk akal secara bisnis dan logis.

📋 Aturan Range
Usia: 16 – 100 tahun
Omzet: Rp 0 – Rp 4,8 Miliar/bulan
Tanggal lahir: tidak boleh di masa depan
NIK: digit ke-7–10 = tanggal lahir valid
❌ Error Message
"Usia Anda kurang dari 16 tahun. Program ini untuk pelaku usaha dewasa."
"Usia melebihi 100 tahun — periksa kembali tanggal lahir Anda."
"Tanggal lahir tidak boleh di masa depan."
SkenarioRespons
Lahir 2015 (usia 9 th)REJECT di bawah 16
Lahir 1900 (usia 124 th)REJECT di atas 100
Lahir 1990 (usia 34 th)PASS
🚫
Test for Invalid Values
V-05

Menolak nilai yang secara semantik tidak valid walaupun format sudah benar — misalnya nilai negatif, nol tidak logis, atau pilihan tidak tersedia.

❌ Nilai yang Ditolak
Omzet < 0 → "Omzet tidak boleh bernilai negatif."
Kategori = kosong → "Pilih kategori usaha yang valid."
Tanggal lahir = hari ini → "Usia 0 tahun tidak valid."
Kode peserta = "TNG-0000" → "Kode TNG-0000 tidak valid."
⚙ Mekanisme Sistem
Field ditandai merah → input direset atau dikunci → pesan error kontekstual ditampilkan → fokus kembali ke field bermasalah.
InputRespons
Omzet = -500000REJECT negatif
Omzet = 0PASS usaha baru
Kode = "TNG-0000"REJECT tidak valid
🗄️
Test for Comparison with Stored Data
V-06

Pengecekan database untuk mencegah duplikasi. Nomor HP dan NIK harus unik — satu nomor hanya untuk satu peserta.

🔍 Prosedur Database
1. Setelah field HP/NIK diisi → sistem mengirim query ke DB
2. Animasi loading ditampilkan ("Mengecek ke database…")
3. Jika sudah ada → tampilkan notifikasi duplikasi
4. Jika unik → field ditandai hijau ✓
❌ Error Duplikasi
"Nomor HP 081234567890 sudah terdaftar atas nama Budi Santoso. Gunakan nomor HP berbeda."

"NIK ini sudah digunakan untuk mendaftar sebelumnya."
SkenarioRespons
HP sudah ada di DBREJECT duplikat
HP belum terdaftarPASS unik
NIK sudah digunakanREJECT duplikat
